Day 5 - TS & Array Headaches

Today I learned...

  • If you'd like to update an array in React, you should avoid any methods that mutate an array (e.g. push(), pop(), splice(), sort()), and instead create a copy of the existing array and then set the state to use that copy.
  • There's some unexpected behavior of React's useState when using TypeScript. For example, if you pass an empty array as an argument into the useState() without specifying its type, it's going to set the type to be an array of never, thus being unable to later edit the state of the array.
  • Quick fix, but I tried render an array of numbers inside my JSX element & discovered that I needed to convert the output to string since you can only render components or strings inside JSX.
