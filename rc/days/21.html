Today I learned...

  • In the context of large language models (LLMs), a token is a unit of text that the model considers the smallest unit of processing per the model's tokenizer. Each tokens has a specificed number of characters, and they are often used as the unit of billing for usage.
  • Temperature is another concept used in LLMs which refers to a parameter which controls the randomness of a model's response, influencing how likely a model is to choose less probable words over more probable ones in its response. For example, a temperature of 0 refers to a model picking the highest-probability token, whereas a temperature of 1 indicates the model picking much less coherent tokens.
  • PUT vs POST are two HTTP methods to send data to a server. I've often used POST requests to send form data to a server, but I was less familiar with the use cases of a PUT request. It is best to decide whether to use a PUT or POST request based on the idempotence of the action. Thus, a PUT request will replace whatever is at a given URL and return the same response regardless of how many requests I make, whereas a POST request will create a resource or cause a change in the server that will return a different response each time. e.g. PUT x=10 vs POST x++;
