Today I learned...

  • An interpreter is a program designed in one language (e.g. Java, C, etc.) to execute code written in another language line by line; this differs from a compiler in that compilers typically translate the entire source code into machine code to create an executable. (although a compiler can simply be used for translating one language into any other).
  • I've used 'this' before, but I got more clarity when solving a quick bug; I called the 'this' keyword in an object nested within another object, and I originally thought it would apply to the parent object, only to remember that 'this' refers directly to the context (i.e. object) in which it is called.
  • Tokens, although used in LLMs, is also a term used in lexical analysis for interpreters; lexical analysis refers to the process of converting raw input text into a sequence of tokens, subsequently allowing for the analysis of the token sequence to build a data structure representing the grammatical structure of the program.
