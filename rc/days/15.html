Day 15 - Game Start, Loop, & End

Today I learned...

  • Packing algorithms are methods used to arrange objects within a container in an efficient way, and are widely used in comptuer science for memory allocation. Types of packing algorithms include the first-fit, best-fit, worst-fit, and next-fit algorithms.
  • A game loop is a continuous loop that runs throughout the life of a game, and it is responsible for updating the game state, rendering, and timing. requestAnimationFrame() is a browser API which is used to schedule a callback function that will be executed before the next repaint of the browser; the callback function also receives timestamp parameter which allows you the calculate the time elapsed between frames and update the animation state.
  • Collision detection is the process of determining when two or more objects in a game interact with each other, and types of collision detection include Bounding Box Collision, Pixel-Perfect Collision, Spatial Partitioning, and Physical Simulation. getClientRect() is a method which provides information about the position and size of an HTML element relative to the viewport, and it can be used in Axis-Aligned Bounding Box collision detection to check if two rectangular areas are overlapping.
